Changes, bug fixes and new programs
TISEAN_1.2 -> TISEAN_2.0

Changes

  • As you may haven noticed, the layout of the manuals has changed quite a bit. In particular, the distinction between a Dresden and a Wuppertal branch is no longer made. Even internally it is softening up.
  • There are no more C++ programs in the package - one compiler less needed!
  • There are no more C programs on the Wuppertal side, but that matters only for Thomas.
  • The internal distinction Wuppertal/Dresden is now replaced by the distinction C/FORTRAN.
  • A number of redundancies between the former Wuppertal/Dresden branches has been removed. We hope this makes life easier for many users. We apologize to those who are missing their choice. The sources of the removed programs are still in the unsupported directory.
  • We adopted a new concept for the Windows executables.
  • The output of the correlation sum programs has been unified. Thus, the post-processing routines are now operational for all such output.
  • Some changes in the modular structure of randomize.
  • Changes to individual programs:
    • addnoise: can also just generate noise
    • ar-model: mean is subtracted first
    • av-d2: input/output modified
    • c1: output modified, doubly visited k's removed
    • c2naive: output modified
    • c2d: input/output modified
    • c2g: input/output modified
    • c2t: input/output modified
    • compare: input/output modified
    • d2: now handles multivariate data as well
    • histogram: re-written in C
    • lyap_k: re-written in C
    • randomize: several new cost functions, some options modified
    • rbf: placement of centers modified
    • stp: improved output
    • surrogates: re-written in FORTRAN. Can now handle multivariate data
    • timrev re-written in FORTRAN
    • xc2: input/output modified

New programs

  • ar-run: Iterate AR model
  • choose: Choose sub-sequence or column
  • cluster: Cluster a dissimilarity matrix
  • endtoend: Determine end-to-end mismatch
  • events: Interval to event time conversion
  • fsle: Finite size Lyapunov exponent
  • intervals: Event time to interval conversion
  • mzeroth: Zeroth order prediction on multivariate time series
  • nstat_z: Nonstationarity test
  • randomize_spikeauto_exp_random: Spike train surrogates (time domain)
  • randomize_spikespec_exp_event: Spike train surrogates (spectral domain)
  • randomize_uneven_exp_random: Surrogates for unevenly sampled data
  • resample: Resample the data
  • spikeauto: Spike train autocorrelation function
  • spikespec: Spectrum of spike trains

No longer supported

The remnants of these goodies are still there, look in the directory unsupported if you desperately need any one of these. But you have to figure out how to compile them yourself.
  • noise was too hard to maintain.
  • autocor3 was not so useful.
  • former delay, now replaced by delay the former embed.
  • make_ps should be obsolete for anyone with a reasonable plotting program at hand.
  • lyapunov was removed due to its redundancy with lyap_k.
  • c2 was removed due to its redundancy with d2.
  • randomize_generic_exp_random was pretty but useless, that is, pretty useless.
